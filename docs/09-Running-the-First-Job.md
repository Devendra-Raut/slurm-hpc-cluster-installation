# 09 - Running the First Job

## Overview

After configuring the Slurm controller, compute nodes, Munge authentication, and MariaDB accounting, the final step is to validate the cluster by submitting and executing a test job.

This chapter demonstrates how to verify the health of the cluster and execute the first Slurm job.

---

# Verify Cluster Status

Before submitting a job, verify that the controller can communicate with all compute nodes.

Display cluster information:

```bash
sinfo
```

Expected result:

* Partition is available.
* Compute nodes are in the **idle** state.
* Cluster is ready to execute jobs.

---

# Display Node Information

View detailed information about the compute nodes.

```bash
scontrol show nodes
```

This command displays:

* Node State
* CPU Count
* Memory
* Available Resources
* Node Address
* Operating System

---

# Display Partition Information

```bash
scontrol show partition
```

Verify that:

* Default partition is configured.
* Compute nodes belong to the correct partition.
* Partition state is **UP**.

---

# Create a Test Job

Create a simple batch script.

```bash
vi hello.sh
```

Example:

```bash
#!/bin/bash
#SBATCH --job-name=hello
#SBATCH --output=hello.out
#SBATCH --ntasks=1

hostname
date
echo "Hello from Slurm!"
```

Save the file.

Grant execute permission.

```bash
chmod +x hello.sh
```

---

# Submit the Job

Submit the batch job.

```bash
sbatch hello.sh
```

Example output:

```text
Submitted batch job 1
```

The returned Job ID uniquely identifies the submitted job.

---

# Monitor the Job Queue

Display all queued and running jobs.

```bash
squeue
```

This command shows:

* Job ID
* User
* Partition
* Job State
* Execution Node

---

# View Job Details

Display detailed information about the job.

```bash
scontrol show job <JobID>
```

Example:

```bash
scontrol show job 1
```

---

# Check Job Accounting

If SlurmDBD is configured, view accounting information.

```bash
sacct
```

This command displays:

* Job Name
* User
* Start Time
* End Time
* Exit Code
* CPU Usage

---

# Verify Job Output

Display the output generated by the job.

```bash
cat hello.out
```

Example:

```text
compute1
Thu Jul 02 10:32:14 IST 2026
Hello from Slurm!
```

This confirms that:

* The job executed successfully.
* A compute node processed the workload.
* Slurm scheduling is functioning correctly.

---

# Useful Commands

| Command             | Description                   |
| ------------------- | ----------------------------- |
| sinfo               | Display cluster status        |
| squeue              | Show running and pending jobs |
| sbatch              | Submit a batch job            |
| srun                | Run an interactive job        |
| scancel             | Cancel a running job          |
| sacct               | View job accounting           |
| scontrol show nodes | Display node details          |
| scontrol show job   | Display job information       |

---

# Validation Checklist

Before proceeding, verify the following:

* Slurm controller is running.
* All compute nodes are online.
* Partition status is **UP**.
* Jobs are accepted by the scheduler.
* Jobs execute successfully.
* Job output is generated.
* Accounting information is recorded.

---

# Outcome

At this stage, the Slurm cluster is fully operational.

Users can:

* Submit batch jobs
* Monitor job execution
* Track resource utilization
* View accounting information
* Execute workloads across multiple compute nodes
